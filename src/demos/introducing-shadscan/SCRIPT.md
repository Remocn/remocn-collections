# SCRIPT — Introducing shadscan

**VO_MODE: none — this video is wordless by design.**

Like every cut in this repo's demo register (and the react-doctor reference it follows), the
spot carries no spoken narration and no BGM: `music: none`. The terminal session, the kinetic
type, and the ascii cover speak; a voiceover would only caption what the product already
shows.

The `voiceover:` fields in STORYBOARD.md are therefore **implied** spoken lines — the shape
each beat is written in (its blueprint's script pattern), kept on the frame as the design
record of what the beat "says". They are locked as on-screen pacing cues, not recorded audio:

1. Hook — "It looks right. It feels done. It ships broken."
2. Intro — "Meet shadscan — deterministic UI audits for shadcn apps."
3. Scan — "One command. Six categories. Every finding with evidence and a fix — and a score."
4. Handoff — "Hand the findings to your agent — evidence, fixes, acceptance criteria.
   Rescan: a hundred."
5. Gate — "The same score on every run — so it can gate your CI. Regressions don't merge."
6. CTA — "One command — pnpm dlx @shadscan/cli@next — and you know. shadscan.com."

If a narrated cut is ever needed, these six lines are the locked narration, already
cue-segmented for the reveal pacing.
