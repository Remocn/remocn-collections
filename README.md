# remocn demos

A gallery of motion design videos generated by AI agents — real React
components built with [remocn](https://remocn.dev) and rendered with
[Remotion](https://remotion.dev). Every video started as a single prompt to a
coding agent; this repo is also a
[shadcn GitHub registry](https://ui.shadcn.com/docs/registry/github), so any
demo can be installed straight into your own Remotion project.

**Gallery:** [demo.remocn.dev](https://demo.remocn.dev) — each demo page shows
the video, the prompt it was generated from, the source code, and the install
command.

## Install a demo

```bash
npx shadcn@latest add kapishdima/remocn-demo/<demo-id>
```

For example:

```bash
npx shadcn@latest add kapishdima/remocn-demo/introducing-remocn
```

This installs the full composition under `demos/<demo-id>/`, pulls its remocn
component dependencies from [remocn.dev](https://remocn.dev), and includes the
original generation prompt as `demos/<demo-id>/prompt.md`. The CLI prints how
to register the `<Composition>` in your Remotion Root after installing.

Demo ids match the gallery slugs — `introducing-remocn`, `paper-shaders`,
`typography`, `agent-skills`, `new-transitions`, and so on. The full list lives
in [`registry.json`](./registry.json).

### Requirements

- A [Remotion](https://remotion.dev) project (`npx create-video@latest`)
- Tailwind v4 wired into Remotion via
  [`@remotion/tailwind-v4`](https://remotion.dev/docs/tailwind) — needed by the
  demos that render real shadcn/ui components inside the video
- Keep all `remotion` and `@remotion/*` packages pinned to the **same exact
  version** — mixed versions break rendering

## Render a video locally

```bash
npx remotion render <demo-id> out/<demo-id>.mp4 --scale=2 --crf=15 --x264-preset=slower --jpeg-quality=95
```

The quality flags render at 2× resolution (720p compositions → 1440p) with a
high-quality encode — dark gradients band badly at the defaults.

Demos that use [Paper shaders](https://paper.design) need the ANGLE OpenGL
backend:

```bash
npx remotion render paper-shaders out/paper-shaders.mp4 --scale=2 --crf=15 --x264-preset=slower --jpeg-quality=95 --gl=angle
```

Each demo page on the gallery shows the exact command.

## Development

```bash
bun install
bun run dev          # Next.js site
npx remotion studio  # Remotion Studio
```

Maintenance scripts:

```bash
node scripts/scan-demos.mjs           # refresh src/demos/demo-meta.json
node scripts/build-registry.mjs       # rebuild registry.json (+ verification log)
node scripts/generate-thumbnails.mjs  # refresh public/thumbnails
node scripts/generate-og.mjs          # refresh public/og (1200x630)
```

Demo assets load from absolute `raw.githubusercontent.com` URLs (see
`src/lib/demo-assets.ts`) so installed demos work without copying this repo's
`public/` directory. Point `REMOTION_DEMO_ASSETS_BASE` at another base URL to
override — e.g. a local static server while working offline.

## How this repo is organized

- `src/demos/<id>/` — one folder per demo: composition, `prompt.md`
- `src/demos/_ui/` — frozen shadcn/ui kit used *inside* videos, pinned to the
  zinc tokens the videos were authored with (the site theme can change freely
  without re-rendering history)
- `src/components/remocn/` — local copies of remocn registry components
- `registry.json` — the shadcn registry; built by `scripts/build-registry.mjs`,
  which diffs each local remocn copy against remocn.dev and emits either a
  `registryDependencies` URL (identical) or bundles the file (diverged)

Built with [remocn](https://remocn.dev). Videos render deterministically — the
gallery player, a local render, and an installed copy produce the same pixels.
